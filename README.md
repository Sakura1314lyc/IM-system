# IM-system

基于 Go 的即时通信系统（Instant Messaging System），支持用户登录、点对点聊天、房间群聊和消息广播。

## 🚀 项目简介

本项目旨在提供一个轻量级、高性能、易扩展的 IM 服务端/客户端原型，帮助学习者掌握：

- WebSocket/Socket 连接管理
- 消息路由与广播
- 用户会话与状态管理
- 多客户端并发处理

> 当前仓库为服务端逻辑 + demo 客户端（如有）

## ✨ 功能特性

- 用户注册/登录认证
- 在线用户列表
- 私聊（点对点消息）
- 群聊房间（Room）
- 消息持久化（可扩展，示例仅内存）
- 断线重连与心跳保活

## 🧩 代码结构

- `main.go` - 应用入口
- `server.go` - IM 服务主逻辑（监听、连接、消息分发）
- `user.go` - 用户管理与状态模型
- `README.md` - 项目说明文档

## 🛠️ 快速运行

1. 克隆仓库

```bash
git clone https://github.com/Sakura1314lyc/IM-system.git
cd IM-system
```

2. 运行服务端

```bash
go run main.go
```

3. 按照现有客户端示例（或 WebSocket 测试工具）连接并测试。可在需要时补充 `client` 模块。

## ⚙️ 配置项（示例）

可根据你项目需求，添加如：

- 监听地址: `:8080`
- 心跳间隔
- 最大连接数
- 日志级别

## 🧪 测试

- 单元测试（如果已覆盖）：

```bash
go test ./...
```

- 手动验证：启动服务后，用两个以上客户端互相发送消息，验证到达。

## 🛠 可能的扩展方向
